ORDER STATUS SYSTEM Workflow, transitions, cancellation, and timeline 6.1 Status Stages & Visual Design # Status Name Badge Color DB Value Description 1 Pending / Order Created Gray pending Default start state. Order exists, no action taken yet. 2 Order Accepted Blue accepted Team has acknowledged and accepted the order. 3 Ready for Delivery Yellow/Amber ready_for_delivery Order is packed and ready to be picked up by delivery. 4 Out for Delivery Orange out_for_delivery Delivery is en route to the customer. 5 Delivered Green delivered Customer has received the order. Final successful state. â€” Cancelled Red cancelled Order was cancelled. Can happen from any stage. 6.2 Status Transition Rules The system allows both forward and backward movement, plus cancellation from any stage: Forward: Any status can advance to the next sequential status Backward: Any status can revert to the previous status (with a warning in UI) Cancel: Available from any status except 'Delivered' â€” shows confirmation dialog Delivered orders: Cannot be cancelled or moved backward â€” they are the terminal success state Cancelled orders: Status is locked â€” no further transitions possible Every transition â€” forward, backward, or cancel â€” creates a new timeline entry in the order_timeline table. No transitions are silent. 6.3 Timeline Entry Structure Each timeline event records exactly who did what and when: order_id: Reference to the parent order from_status: What the status was before (null for initial creation) to_status: What the status changed to changed_by_user_id: Reference to users table changed_by_name: Denormalized user name for display performance note: Optional text remark added during status change created_at: Exact timestamp with timezone 7.
